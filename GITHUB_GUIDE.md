## 1. Что такое зелёные иконки

Зелёные квадратики на главной странице GitHub — это **график вкладов
(contribution graph)**. Каждый квадрат = 1 день. Чем больше коммитов
(сохранений) в этот день, тем темнее зелёный.

### ✅ Что засчитывается в зелёные иконки

| Действие | Условие |
|----------|---------|
| **Commit** в репозиторий | Репозиторий должен быть **публичным** ИЛИ вы делаете коммит в **форк** публичного |
| **Pull Request** (PR) | PR должен быть **слит (merged)** в публичный репозиторий |
| **Issue** | В публичном репозитории |
| **Star** репозитория | ❌ НЕ считается |

### ⚠️ Три золотых правила, чтобы коммиты засчитались

1. **Email в git = Email на GitHub.** Это самая частая причина «пропадающих»
   коммитов. Если email не совпадает — коммит есть, а зелёного квадратика нет.
2. Коммит должен быть в ветку **`main`** или **`master`** (по умолчанию) либо в
   ветку, из которой позже делается PR в `main`.
3. Репозиторий должен быть **публичным** (Public), а не приватным.

## 2. Подготовка

### 2.1. Проверьте свой email на GitHub

1. Зайдите на https://github.com/settings/emails
2. Запомните основной email (например, `burkin.anton@bk.ru`).

### 2.2. Проверьте email в git

Откройте PowerShell в папке проекта и выполните:

```powershell
git config user.name
git config user.email
```

**Если email НЕ совпадает с GitHub** — исправьте (замените на свой GitHub-email):

```powershell
git config --global user.name "Antosha122"
git config --global user.email "burkin.anton@bk.ru"
```

> 💡 Хотите скрыть реальный email? На https://github.com/settings/emails
> включите «Keep my email addresses private» и используйте специальный адрес
> вида `12345678+Antosha122@users.noreply.github.com` — он тоже засчитывается.

### 2.3. Секреты уже защищены ✅

Перед публикацией в проекте закрыты все утечки:

| Файл | Защита |
|------|--------|
| `.env` (токены ботов, пароль БД) | В `.gitignore`, не попадёт в git |
| `.env.example` | Пароль заменён на `your_secure_password_here` |
| `tokio_bar_backup.sql` (дамп БД) | В `.gitignore` (`*_backup.sql`) |
| `docker-compose.yml` | Секреты берутся из переменных, не захардкожены |

**Дополнительно: смените пароль PostgreSQL** `122xf34A` → новый, т.к. старый
мог быть в `.env.example` ранее. Также перевыпустите токены ботов у
[@BotFather](https://t.me/BotFather), если есть сомнения.

## 3. Создание репозитория

### Шаг 1. Создайте репозиторий на GitHub

1. Откройте https://github.com/new
2. **Owner:** `Antosha122` (ваш аккаунт)
3. **Repository name:** например, `tokio-bar-bot`

После создания GitHub покажет адрес вида:


Запомните его — используем дальше.

## 4. Заливка проекта

Выполните в PowerShell **в папке проекта**
(`f:\Work\pythonProject(QR коды рестораны)`):

### 4.1. Инициализация git

```powershell
cd "f:\Work\pythonProject(QR коды рестораны)"

# Инициализируем репозиторий
git init -b main

# Проверяем, что секреты НЕ попадут в индекс (должно быть пусто)
git status
```

> ⚠️ Если в `git status` виден `.env` или `tokio_bar_backup.sql` — остановитесь
> и проверьте `.gitignore`. Они должны быть проигнорированы.

### 4.2. Первый коммит

```powershell
# Добавляем все файлы (кроме тех, что в .gitignore)
git add .

# Проверяем, что добавилось — ещё раз убедитесь, что НЕТ .env
git status

# Создаём первый коммит (он даст вам первый зелёный квадратик! 🟩)
git commit -m "Initial commit: Tokio Bar restaurant bot system"
```

### 4.3. Привязка к GitHub и отправка

```powershell
# Привязываем локальный репозиторий к GitHub (ЗАМЕНИТЕ URL на свой!)
git remote add origin https://github.com/Antosha122/tokio-bar-bot.git

# Отправляем код на GitHub
git push -u origin main
```

При первом push GitHub запросит логин и пароль. **Вместо пароля используйте
Personal Access Token (PAT)** (см. ниже), либо войдите через `gh auth login`.

### 4.4. Создание Personal Access Token (если нужен)

1. Откройте https://github.com/settings/tokens?type=beta (fine-grained tokens)
2. **Generate new token** → выберите репозиторий `tokio-bar-bot`
3. В Permissions → **Repository permissions** → **Contents: Read and write**
4. Скопируйте токен (вида `github_pat_xxx...`)
5. При `git push` введите логин `Antosha122` и пароль = этот токен.

> 💡 Проще всего — установить [GitHub CLI](https://cli.github.com/) и один раз
> авторизоваться: `gh auth login`. После этого push будет работать без паролей.

### 4.5. Проверка

1. Откройте `https://github.com/Antosha122/tokio-bar-bot`
2. Убедитесь, что файлы загрузились, а `.env` и `.sql` — **отсутствуют**.
3. Зайдите в свой профиль `https://github.com/Antosha122` — увидите зелёный
   квадратик за сегодняшний день! 🎉

## 5. Ежедневная работа

Чтобы зелёные иконки появлялись регулярно, работайте по циклу:

### Стандартный рабочий цикл

```powershell
# 1. Перед началом работы — заберите свежие изменения
git pull

# 2. Редактируете код (в VS Code или другом редакторе)...

# 3. Смотрите, что изменилось
git status
git diff

# 4. Добавляете изменённые файлы
git add .

# 5. Коммит с понятным сообщением (что и зачем изменили)
git commit -m "Добавил категорию напитков в меню"

# 6. Отправляете на GitHub
git push
```

### 💡 Как делать «осмысленные» коммиты каждый день

Не обязательно писать сотни строк кода каждый день. Полезные мелкие задачи:

| Задача | Пример коммита |
|--------|----------------|
| Новый пункт меню | `Добавил ролл «Филадельфия» в меню` |
| Исправление бага | `Фикс: корзина не очищалась после заказа` |
| Документация | `Обновил README: добавил раздел FAQ` |
| Рефакторинг | `Разделил staff_handlers на функции` |
| Тесты | `Добавил тест на авторизацию официанта` |
| Комментарии | `Добавил docstring к OrderService` |

Главное — **делайте коммит в тот же день**, когда работали. GitHub считает дату
коммита, а не дату push.

### Ветки и Pull Requests (продвинутый уровень)

Для крупных изменений лучше использовать ветки:

```powershell
# Создаём новую ветку для фичи
git checkout -b feature/delivery-module

# ...работаете, коммитите...
git add .
git commit -m "Добавил модуль доставки"

# Отправляете ветку
git push -u origin feature/delivery-module
```

Затем на GitHub откройте **Pull Request** и слейте его в `main`. Слитый PR
тоже даёт зелёный квадратик (даже два: за коммит + за PR).

## 6. Проверка

### Если зелёные квадратики НЕ появляются — чек-лист

1. **Email совпадает?**
   ```powershell
   git config user.email   # должен совпадать с email на GitHub
   ```
   Проверить, какой email у конкретного коммита:
   ```powershell
   git log -1 --format="%ae"
   ```

2. **Репозиторий публичный?**
   Settings → в самом низу «Danger Zone» → Change repository visibility →
   должно быть **Public**.

3. **Коммит в `main`/`master`?**
   Коммиты в другие ветки засчитываются, только если позже слиты через PR.

4. **Это форк?**
   Коммиты в форк засчитываются только если форк публичный и коммит в его
   `main`/`master`.

5. **Дата коммита в прошлом?**
   GitHub показывает вклады по дате коммита. Если «откатить» дату — квадратик
   появится в прошлом, а не сегодня.


### Первый push (один раз)

```powershell

git init -b main
git add .
git status          # проверить, что .env НЕТ в списке!
git commit -m "Initial commit: Tokio Bar restaurant bot system"

git push -u origin main
```

### Каждый день (рабочий цикл)

```powershell
cd "f:\Work\pythonProject(QR коды рестораны)"
git pull                          # забрать свежее
# ...редактируем код...
git add .                         # добавить изменения
git commit -m "Что сделал: ..."   # закоммитить (дата = сегодня → зелёный квадратик)
git push                          # отправить на GitHub
```

### Полезные команды

```powershell
git status                  # что изменилось
git log --oneline -10       # последние 10 коммитов
git diff                    # несохранённые изменения
git diff --staged           # изменения в индексе (после git add)
git checkout -- .           # отменить все локальные изменения (ОСТОРОЖНО!)
git revert HEAD             # отменить последний коммит (безопасно, создаёт новый)
```

## ❓ Частые вопросы

**В: Можно ли сделать много коммитов за один день?**
О: Можно, но в графике отобразится один квадратик (потемнее). Для видимого
роста важна **регулярность по дням**, а не количество в один день.

**В: Я случайно запушил `.env` с токенами! Что делать?**
О: 1) Немедленно перевыпустите токены у BotFather и смените пароль БД.
2) Удалите файл из истории:
```powershell
git rm --cached .env
git commit -m "Удалил .env из отслеживания"
git push
```
Для полной очистки истории используйте `git filter-repo` или BFG Repo-Cleaner.

**В: Гитхаб просит пароль при push.**
О: Пароль не подойдёт. Нужен **Personal Access Token** (раздел 4.4) или
авторизация через `gh auth login`.

**В: Зелёные иконки появились не сразу.**
О: График обновляется с задержкой до нескольких минут. Если через сутки нет —
проверьте email (раздел 6, пункт 1).
